Requirements Document

**Functional** **Requirements**

**Login & Sign Up**:

- The website must contain a login in link to allow the user to log in
  to their personal account

- The website should have a forgot password link to help the user log in

- The website must contain a sign-up link to allow the user to sign in
  to their account

- The website will use a database to store the accounts and account
  details

> **Progress and Level:**

- The website must allow the user to make progress while learning a
  subject through passing tests and quizzes

- The progress must be displayed to the user based on their subject

- The user will not be able to skip a level or reach higher progress in
  a subject without progressing through the levels of that subject

- The website will use a database to store user progress and level

> **Customized Learning:**

- The user must be able to choose a learning path and subject to begin a
  game or start learning (will implement elementary math learning path)

- A learning path must be chosen first, all subjects will be contained
  in a learning path (will implement addition, multiplication, and
  division, and fractions subjects for math learning path)

- Learning path progress will be presented even if the user does not
  wish to continue entire learning path

- Any subject in a learning path can be accessed but the levels in the
  subject must be reached through passing the preceding level

> **Learning Mode:**

- The user will be able to choose to learn through games or without
  games

- The game learning mode will feature same learning practices as the
  non-game mode

- The game mode will have games as tests and games as quizzes

- The games will not require skill, only an understanding of the
  material

- The traditional learning mode will feature traditional learning
  practices on screen and traditional tests and quizzes on screen

> **User Score & Problem Difficulty:**

- Users will be given individual scores based on their progress for a
  specific learning path

- The user's total score will be the sum of the scores earned from each
  path

- More difficult learning paths will have more lessons and will result
  in higher scores upon completion

> **Game Details:**

- Games may be designed using Unity game engine

- Games will consist of a playable character that the user controls

- One game style will have the user attempting to reach a target within
  a time limit to win but will be asked questions to advance, if the
  answer is incorrect or the user takes too long to answer the user will
  be set back and if time runs out the user will lose and have to try
  again (racing or fighting game)

- Another game style will have the user trying to accomplish a task but
  in order to be able to gather the materials to accomplish the task
  they must correctly answer questions, if the questions are not
  answered correctly in a time limit the user will lose and have to
  restart (escape room game)

**Look and Feel**

**Appearance**

The website should have a traditional look and feel. The homepage should
have a header with the website name and logo. It will display an array
of games and subjects that are available to be used in the middle of the
page. There should also be links to pages for the user to login or
create an account. No website features are available until the user is
in an account. The website will then display user progress at the top
and all the learning paths that can be selected underneath as well as
their individual progress. There will be a community link where users
can see the scores of other users.

**Usability**

The completed website should be easy for users to navigate and access
desired functions. It will consist of clearly labeled menus and links to
certain features. The overall functionality of the website should be
simple to understand and easy to learn for beginners. The website may
allow simple customization such as light mode and dark mode features to
match desired viewing preferences of the user. The website should be
hosted by a reliable provider with a server that ensures the website is
always accessible and running properly.

**Performance**

- Games should be written in a high-performance language to ensure that
  fastest runtime speeds are possible

- The website should have a server that guarantees appropriate speeds
  for the expected amount of traffic

- The website should be well designed and properly documented to ensure
  that it can be extended and updated with new features easily

- Because the website will contain files of information to help users
  learn, the data must be delivered to users without error

- Data will need to be accurate arriving to the user and user data will
  need to be kept safe while making sure that it is still accessible in
  a reasonable amount of time

- The database will need to be created for efficient and fast retrieval
  of data, and it will need to consist of necessary storage for future
  expansion of users

**Maintainability and Support**

- The database will be implemented to be secure

- The website will be implemented to be reliable and responsive

- Functionality updates will be coded as they become available, and
  updates will be made from user requests given that they are currently
  possible

**Security**

- Necessary front-end security to ensure users cannot enter malicious
  data on login and sign-up forms

- Necessary backend security to further validate user data and prevent
  unauthorized access

- Encrypting passwords and other sensitive information to store in
  database

- Protecting personal passwords to prevent unauthorized access to the
  database

**Cultural**

- Users will not have to use a mouse to play the games, games will be
  played using the keyboard only (games will be accessible)

- English will be the language of the page controls, but games can be
  understood and played by anyone

- No sensitive content that some people may find offensive will be
  displayed on the website
